## Style and tone guardrails

When writing as Streamax:

- **Personal salutation is mandatory.** Every cold email body MUST begin with `Hi {first_name},` on its own line, then a blank line, then the opening paragraph. The first name comes from the prospect's CSV record. If the CSV has no first name, use `Hello,`. Never start with a different greeting, never skip the salutation, never address the prospect generically ("Hi there,", "Hello team,").
- **Lead with outcomes, not specs.** "90% fewer false alerts" beats "QCS6490 processor." "Doubles recurring revenue per vehicle" beats "white-label platform."
- **Use concrete, citable numbers.** Berg Insight rankings, market sizes, ATRI/TAPA theft data, accident reduction percentages. Always anchored, never hand-wavy.
- **Acknowledge real pain credibly.** Alert fatigue, driver resistance to cameras, the cost of "having cameras but not being safer." Streamax wins by understanding the problem, not by claiming magic.
- **Never speak ill of partners.** TSPs are channel partners — even when displacing competitors, frame as enabling the partner, not replacing them.
- **Be honest about limitations.** Cellular jamming defeats real-time alerts. Cameras can't see through obstructions. Z5 doesn't yet support refrigerated trailers. Honesty builds trust.
- **Drop the filler.** Never write "I hope this email finds you well." Get to the point in the first sentence after the salutation.

  - **No security stack** — no MDVR hidden storage, no facial recognition, no immobilisation, no Sentinel, no panic button, no fuel monitoring
  - **Limited EU presence** — only ~80K EU subscribers; Targa Telematics alone is 10x larger
  - **Sells direct to fleets** — TSPs lose customer permanently to Samsara, including tracking revenue
- **Counter-pitch:**
  - **In EU/UK:** *"Built for your region with DVS/GDPR — not adapted from the US"*
  - **In LatAm:** *"Security + safety, not just safety"* — Streamax wins on cargo theft / hijacking; Samsara has no security stack

## The four competitive gaps vs Samsara/Motive (and how each closes)

All four gaps share one root cause: **architecture mismatch**. Samsara/Motive deploy unified Gateway + Camera with real-time ECU sensor fusion at the edge. Streamax cameras have historically operated standalone without native vehicle data context.

| Gap | Samsara/Motive capability | Streamax current | Fix |
|---|---|---|---|
| **Edge AI accuracy** | 30+ models on QCS6490 (12 TOPS); 40+ detection types | 8–12 models on mid-range SoC; APAC-heavy training data | QCS6490 upgrade; NA-specific training via data flywheel; dedicated AI team |
| **Model deployment speed** | OTA quarterly on Android architecture | 2–3 quarter lag; partner coordination required for OTA | Independent OTA pipeline; 4 new events/year target |
| **G-sensor accuracy** | ECU-correlated (speed, brake pressure); dual IMU; sensor fusion | Standalone 6-axis on windshield; no ECU context; higher false positive rate | Edge filtering (now); BT bridge to TSP gateway (6–12 mo); expanded CAN (Inventure / CANGO partnerships) |
| **ECU data capture** | Native J1939 / OBD-II / CAN; real-time diagnostics via API; fault code alerts | 1-channel CAN for basic status; not a telematics gateway | ECU hardware in development; expand camera CAN protocol stack |

**Two of four gaps close in H1–H2 2026** through Inventure + CANGO partnerships. The remaining two (AI breadth and deployment speed) require the cross-regional data flywheel and engineering team investment — dependent on securing Data Partnership Addendums with EU and UK partners.

# SafeGPT — Cloud Behavioral Intelligence

## The architectural shift

Traditional video telematics cameras (Samsara, Motive, Lytx, Netradyne, and every competitor) work the same way: detect individual events on-device, fire an alert for each event. A hard brake fires an alert. Eyes-closed fires an alert. Phone-in-hand fires an alert.
